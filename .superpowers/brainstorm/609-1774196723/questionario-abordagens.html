Como implementar o questionário?

3 abordagens possíveis — com trade-offs

A ⭐

Server Actions + Resend + Vercel Blob

Recomendado. Wizard client-side com React state. Cada etapa é um componente. No envio, um Server Action chama Resend (e-mail) e Vercel Blob (uploads). Tudo dentro do Next.js, sem nova API. Idiomático com o resto do projeto.

  • ✅ Zero backend extra — usa o que já existe
  • ✅ Upload seguro (assinado pelo servidor)
  • ✅ Fácil de adicionar i18n via next-intl
  • ⚠️ Requer conta Resend e Vercel Blob
B

React Hook Form + API Route

Wizard com react-hook-form para validação robusta. Rota /api/questionnaire separada para lidar com o envio. Mais boilerplate mas mais controle granular por campo.

  • ✅ Validação de formulário muito poderosa
  • ✅ Melhor para formulários com lógica condicional complexa
  • ⚠️ Mais arquivos, mais setup
C

Typeform / Tally embed

Ferramenta externa embutida via iframe. Zero desenvolvimento para os campos. Layout bonito por padrão. Porém fora do controle do design do site.

  • ✅ Rápido de colocar no ar
  • ❌ Visual não combina com o site
  • ❌ Custo mensal para features avançadas
  • ❌ Dados dos clientes em terceiro
